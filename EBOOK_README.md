## 电子书本地阅读（v1.2 新增）

**当用户说「我在 D 盘电子书文件夹下载到了这本书」或明确告知本地电子书路径时，优先用本地文件读取，不要用在线搜索。**

### 支持的格式
| 格式 | 依赖 | 状态 |
|------|------|------|
| `.epub` | 内置 `zipfile`（无需额外安装） | ✅ 可用 |
| `.pdf` | `pip install pypdf` | ✅ 可用 |

---

### 快速使用

```powershell
# 1. 列出全书章节
cd C:\Users\Administrator\.qclaw\skills\book-learning-assistant\scripts
python3 read_book.py "D:\电子书\书名.epub" --list

# 2. 读取指定章节（关键词 = 章节标题）
python3 read_book.py "D:\电子书\书名.epub" "第2章 标题" --limit 15000
```

返回 JSON 格式：
```json
{
  "file": "text/part0007.html",
  "chars": 5716,
  "text": "第2章 更接近现实\n\n星系是大得令人难以置信的物体...",
  "epub": "D:\\电子书\\书名.epub"
}
```

### 工作流程（集成到读书技能）

1. **用户告知本地电子书路径**
   ↓
2. **调用 `read_book.py --list` 获取目录**
   ↓
3. **调用 `read_book.py "关键词"` 读取指定章节原文**
   ↓
4. **基于原文撰写学习笔记**（不再靠搜索推断）
5. 写入 Obsidian + Notion

---

### 章节匹配逻辑（重要）

脚本通过**三层过滤**定位章节正文，避免误匹配目录/索引页：

| 过滤层 | 规则 | 目的 |
|--------|------|------|
| 1. 长度过滤 | 排除 < 2000 字的文件 | 跳过目录页、索引页 |
| 2. 关键词密度 | `出现次数 × 1000 / 文件长度` | 正文密度 > 索引页 |
| 3. 取密度最高者 | 排序后取第一名 | 精准命中章节正文 |

**测试结果**（《无穷的开始》第2章）：
- `part0002.html`（目录页）→ 415字，**被长度过滤排除** ✅
- `part0025.html`（索引页）→ 17621字，密度 0.2，**密度低，未选中** ✅
- `part0007.html`（第2章正文）→ 5716字，密度 0.5，**密度最高，正确命中** ✅

---

### 注意事项

- EPUB 是 ZIP 格式，脚本通过 `zipfile` 模块直接解析，**不需要解压**
- 若关键词匹配到多个文件，脚本返回**关键词密度最高**的那个
- `--limit` 默认 8000 字符，章节内容较长可设为 15000+
- 如果 `read_book.py` 报错，检查电子书是否完整（有些盗版 EPUB 缺少 `toc.ncx`）
- PDF 格式依赖 `pypdf`，首次使用需运行 `pip install pypdf`

---

### 与在线搜索的对比

| 方式 | 优点 | 缺点 |
|------|------|------|
| **本地 EPUB**（`read_book.py`） | 原文精准、无版权风险、可离线 | 需要提前下载电子书 |
| **在线搜索**（搜索引擎） | 无需本地文件 | 内容碎片化、准确性低、受限于搜索结果 |

**结论**：当用户提供了本地电子书路径时，**一律优先用本地文件**。
